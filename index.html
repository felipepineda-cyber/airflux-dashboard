⏱️ Tiempos de alerta

Se guardan automáticamente (local + nube)
Los sensores envían promedios cada 1 h. Con 2 h se detectan fallas reales evitando falsas alarmas por retrasos.
Evita que llegue el mismo aviso una y otra vez. Al volver el sensor, el contador se reinicia.
☁️ Estos valores se guardan en Supabase (tabla config_alertas): la página y el vigilante 24/7 usan la misma configuración.

💬 Avisos de no funcionamiento — WhatsApp y correo

Enviar avisos automáticos cuando un sensor deje de registrar datos
Máximo 1 aviso por sensor cada 24 h, mientras esta página esté abierta. Para vigilancia 24/7 usa el vigilante de Supabase.
Se hará POST con JSON: {mensaje, whatsapps, correos, origen}. Aquí conectas Twilio o la API oficial de Meta.
📱 Números de WhatsApp (puedes agregar varios)
Cada número debe activar CallMeBot una vez (pasos abajo) y tiene su propia API key.
Enviar también el mismo aviso por correo electrónico
Vía FormSubmit (gratis, sin registro). ⚠️ El primer envío a cada correo llega como solicitud de confirmación: ábrela y presiona "Activate" una sola vez.
📧 Correos de destino (puedes agregar varios)

🛠️ Cómo activar CallMeBot (una sola vez por cada número)

1Desde CADA número que quieras recibir avisos: agrega +34 623 78 95 90 a los contactos y envíale por WhatsApp: "I allow callmebot to send me messages".
2El bot responde a cada número con su API key personal. Agrégala arriba junto a su número con "➕ Añadir número".
3Para los correos: actívalos con su interruptor, agrégalos y envía una prueba; el primer mensaje de FormSubmit llega como solicitud de confirmación — presiona "Activate" en ese correo y los siguientes llegarán normal.
ℹ️CallMeBot es un servicio gratuito de terceros, útil para partir. Para producción (varios destinatarios, sin depender de la página abierta) conviene la fase 2: un backend que vigile los sensores 24/7 y envíe por la API oficial de WhatsApp (Meta) o Twilio — también correo. La opción "Backend propio" de arriba ya deja lista esa conexión.
